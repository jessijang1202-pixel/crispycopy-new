## 1. 신청자 상황 요약

**만들고 있는 앱**: 1인 창업자·프리랜서 마케터를 위한 SNS 콘텐츠 자동 생성·발행 플랫폼

**현재 상태**
- 온보딩 화면, 대시보드, 콘텐츠 발행 화면 껍데기 완성
- 샘플 콘텐츠 생성 테스트 완료
- 로그인 기능 미완성

**수업 전까지 준비할 것**
- 브랜드 DNA 수집 폼 (공통 질문 5~7개 + 직업/업종별 추가 질문)
- 일정 관리 화면 (캠페인/이벤트/일상 스케줄)
- 로그인 기능

**수업 중 해결하고 싶은 기술 문제**
- Claude API 연결 방식 (비용 최적화, 효율적 구조)
- 카톡 자동 발송 연동 (승인 프로세스)
- Claude의 엉뚱한 안내를 피하고 정확한 구현 방법 학습
- 저비용·효율적 배포 방식

**1주차 목표**: 초기 고객 1~2명으로 브랜드 DNA 폼 + 일정 관리 화면 검증  
**2주차 목표**: 피드백 반영해 앱 완성도 높이기

## 2. 오늘 수업에서 만들 AI 직원의 역할

**입력**: 고객이 제공하는 정보
- 브랜드 정보 (공통 항목)
- 직업/업종 선택
- 업종별 추가 질문 답변
- 캠페인/이벤트/일상 일정

**처리**: Claude API를 통한 콘텐츠 생성
- 브랜드 DNA 기억 (톤, 스타일, 가치관)
- 일정 기반 콘텐츠 기획
- 6개 채널별 특성에 맞춘 글 작성

**출력**: 채널별 맞춤 콘텐츠 + 승인 프로세스
- 블로그, 인스타그램, 스레드, 틱톡, 당근, 카톡채널 각각의 글
- 매주 월요일 카톡으로 자동 발송
- 고객 승인 후 자동/반자동 발행

## 3. MVP 범위 (오늘 6시간 안에)

### Phase 1: 핵심 기능만 구현

**1단계: 브랜드 DNA 수집 폼 설계 완성**
- 공통 질문 5~7개 예시
  - 브랜드명
  - 한 줄 설명 (타겟 고객이 느껴야 할 감정)
  - 주요 제품/서비스
  - 브랜드 톤 (친근함/전문성/고급스러움 등)
  - 타겟 고객층
  - 경쟁사 대비 차별점
  - 주요 메시지 3가지

- 직업/업종별 추가 질문 (3~5개 유형만 먼저)
  - 예: 1인 창업자 / 프리랜서 / 소상공인 / 기타
  - 각 유형별 3~4개 추가 질문

**2단계: 일정 관리 화면 구현**
- 캠페인 일정 입력 (시작일, 종료일, 캠페인명, 핵심 메시지)
- 이벤트 일정 입력 (날짜, 이벤트명, 설명)
- 일상 콘텐츠 (특정 일정 없음)
- 간단한 캘린더 또는 리스트 뷰

**3단계: Claude API 연결 (한두 채널 샘플)**
- 블로그 + 인스타그램 2개 채널만 먼저
- 입력된 브랜드 DNA + 일정을 Claude에 전달
- 각 채널 특성에 맞춘 콘텐츠 생성 프롬프트 작성
- 생성된 콘텐츠를 화면에 표시

**4단계: 카톡 자동 발송 (승인 프로세스)**
- 생성된 콘텐츠를 카톡 메시지로 고객에게 발송
- 고객이 "발행" 체크박스로 승인
- 승인된 콘텐츠만 자동/반자동 발행 준비

### 오늘 만들지 않는 것 (Phase 2 이후)

- 모든 6개 채널 동시 구현 (2개만 먼저)
- 실제 SNS 자동 업로드 (반자동 복사·붙여넣기 단계까지만)
- 콘텐츠 저장/이력 관리
- 결제 기능
- 외부 배포 (로컬 테스트만)

```
[고객 입력]
  ↓

  ↓
[시스템 처리]
  ↓
Claude API에 프롬프트 전달
(브랜드 DNA + 일정 + 채널별 특성)

  ↓
[출력 및 승인]
  ↓
카톡으로 콘텐츠 + 일정 전송
  ↓
고객 승인 (체크박스)
  ↓
자동/반자동 발행 준비 완료
```

## 5. 기술 구현 가이드

### Claude API 연결 (비용 최적화)

**주의사항**
- API 키는 절대 코드에 직접 입력하지 말 것
- `.env` 파일에 저장하고 `.gitignore`에 추가
- 환경변수로 읽어오기

**비용 최적화 전략**
- 한 번에 여러 채널 콘텐츠를 한 번의 API 호출로 생성 (개별 호출 X)
- 프롬프트 캐싱 활용 (브랜드 DNA는 반복되므로)
- 모델 선택: `claude-3-5-sonnet` (가성비 최고)

**프롬프트 구조 예시**
```
[시스템 역할]
당신은 SNS 마케팅 전문가입니다.

[브랜드 DNA 정보]
- 브랜드명: {brand_name}
- 톤: {tone}
- 타겟: {target}
...

[일정 정보]
- 캠페인: {campaign_name} ({start_date} ~ {end_date})
- 핵심 메시지: {message}

[요청]
위 정보를 바탕으로 다음 채널별 콘텐츠를 생성하세요:
1. 블로그 (500자, 전문성 강조)
2. 인스타그램 (해시태그 포함, 시각적 설명)

[출력 형식]
JSON으로 반환:
{
  "blog": "...",
  "instagram": "..."
}
```

### 카톡 자동 발송 (카톡 채널 API)

**필요한 것**
- 카톡 채널 API 키 (카카오 비즈니스 계정)
- 고객의 카톡 채널 ID 또는 전화번호

**구현 순서**
1. 카톡 채널 API 문서 확인 (카카오 개발자 센터)
2. 메시지 템플릿 설정 (승인용 메시지 형식)
3. 생성된 콘텐츠를 템플릿에 맞춰 발송
4. 고객 응답 수신 (승인/거절)

**주의**
- 카톡 발송은 고객 동의 필수
- 메시지 발송 시간 제한 확인 (야간 발송 금지 등)

### 데이터 저장 (MVP 단계)

**오늘은 저장하지 않음** — 로컬 메모리 또는 임시 변수로만 처리

**Phase 2에서 추가**
- Notion API 또는 간단한 DB (SQLite, Firebase)
- 고객별 브랜드 DNA 저장
- 생성된 콘텐츠 이력 관리
